# Reverse-Engineered Qwen API

Endpoints discovered on `chat.qwen.ai`:

## 1. Authentication

### Sign In
```http
POST https://chat.qwen.ai/api/v2/auths/signin
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "<sha256-hashed>",
  "login_type": "email"
}
```

Response:
```json
{
  "success": true,
  "data": { ... }
}
```

Error:
```json
{
  "success": false,
  "data": {
    "code": "Bad_Request",
    "details": "The email or password provided is incorrect..."
  }
}
```

The password is **SHA256-hashed client-side** before sending:
```typescript
const hashedPassword = crypto.createHash('sha256')
  .update(password).digest('hex');
```

## 2. Chat Completions (The Main Event)

```http
POST https://chat.qwen.ai/api/v2/chat/completions?chat_id=<session_id>
Content-Type: application/json
bx-ua: <anti-bot fingerprint>
bx-umidtoken: <device token>
bx-v: 2.5.36
cookie: <session cookies>
```

### Request Body
```json
{
  "stream": true,
  "version": "2.1",
  "incremental_output": true,
  "chat_id": "session-uuid-or-null",
  "chat_mode": "normal",
  "model": "qwen3.6-plus",
  "parent_id": "prev-response-id-or-null",
  "messages": [
    {
      "fid": "message-uuid",
      "parentId": null,
      "childrenIds": [],
      "role": "user",
      "content": "Full prompt text here...",
      "user_action": "chat",
      "files": [],
      "timestamp": 1716388800,
      "models": ["qwen3.6-plus"],
      "chat_type": "t2t",
      "feature_config": {
        "thinking_enabled": true,
        "output_schema": "phase",
        "research_mode": "normal",
        "auto_thinking": false,
        "thinking_mode": "Thinking",
        "thinking_format": "summary",
        "auto_search": false
      },
      "extra": {
        "meta": { "subChatType": "t2t" }
      },
      "sub_chat_type": "t2t",
      "parent_id": null
    }
  ],
  "timestamp": 1716388801
}
```

### SSE Response

```
data: {"response.created":{"response_id":"qwen-xxxx"}}

data: {"choices":[{"delta":{
  "phase": "thinking_summary",
  "extra": {
    "summary_thought": {
      "content": ["First thought...", "Second thought..."]
    }
  }
}}]}

data: {"choices":[{"delta":{
  "phase": "answer",
  "content": "Cumulative or incremental text..."
}}]}

data: {"choices":[{"delta":{
  "phase": "answer",
  "content": "...continued text..."
}}]}

data: {"usage":{"output_tokens": 123, "input_tokens": 45}}

data: [DONE]
```

### Phase Types

| Phase | Meaning | Contains |
|---|---|---|
| `thinking_summary` | Model is reasoning | `extra.summary_thought.content[]` — array of thought strings |
| `answer` | Generating output | `content` — text (cumulative OR incremental) |

### The "incremental_output" Flag

When `incremental_output: true` is set, Qwen **sometimes** returns only the delta (new text), and **sometimes** returns the full cumulative text. The proxy handles this with `getIncrementalDelta()` which tries prefix matching first, then falls back to treating it as incremental.

## 3. Models List

```http
GET https://chat.qwen.ai/api/models
```

Response:
```json
{
  "data": [
    {
      "id": "qwen3.6-plus",
      "owned_by": "qwen",
      "info": { "created_at": 1716000000 }
    },
    {
      "id": "qwen3.7-max",
      "owned_by": "qwen"
    },
    {
      "id": "qwen3.6-max-preview",
      "owned_by": "qwen"
    }
  ]
}
```

The proxy appends `-no-thinking` variants for each model and caches results for 1 hour.

## 4. Disable Native Tools

```http
POST https://chat.qwen.ai/api/v2/users/user/settings/update
Content-Type: application/json

{
  "tools_enabled": {
    "web_extractor": false,
    "web_search_image": false,
    "web_search": false,
    "image_gen_tool": false,
    "code_interpreter": false,
    "history_retriever": false,
    "image_edit_tool": false,
    "bio": false,
    "image_zoom_in_tool": false
  }
}
```

This is called once on first header interception to disable Qwen's built-in tools (which would interfere with the proxy's own tool system).

## 5. User Settings

```http
GET https://chat.qwen.ai/api/v2/users/user/settings
```

Used to read current user settings.

## Tokens and Headers

### `bx-v`: `"2.5.36"`
Seems to be a version identifier for the anti-bot system. Hardcoded fallback in the code.

### `bx-ua`
A browser/user-agent fingerprint generated by Qwen's JavaScript. Required for all API calls.

### `bx-umidtoken`
A device identification token. Stored in browser storage.

### `x-request-id`
A UUIDv4 generated fresh for each request. Used for request tracing.
